# Story 3-5: AI 内容来源标注与基线交叉验证 — UX 设计规格

## 1. 功能概述

本 Story 为 AI 生成的方案内容提供"可信层"——在编辑器中为每段 AI 生成内容标注来源类型，并与产品基线进行交叉验证。用户可快速判断每段内容的可信度，防止 AI 编造技术参数。

## 2. 信息架构

```
ProjectWorkspace (三栏布局 + SourceAttributionContext)
├── LeftPanel: DocumentOutlineTree
├── CenterPanel: PlateEditor (主编辑区)
│   ├── ChapterSection
│   │   ├── HeadingElement (章节标题 + 生成按钮)
│   │   ├── ParagraphElement (段落内容)
│   │   │   ├── [装饰层] SourceAttributionLabel (来源标注标签)
│   │   │   │   └── SourceDetailPopover (点击展开详情)
│   │   │   └── [装饰层] BaselineMismatchMarker (基线不匹配标记)
│   │   └── ChapterGenerationProgress (生成进度 + 来源标注阶段 + 可选基线验证槽位)
│   └── ...
└── RightPanel: AnnotationPanel / PropertiesPanel
```

## 3. 核心交互模式

### 3.1 来源标注标签 (SourceAttributionLabel)

**位置**: 每个 AI 生成段落的右侧边距区域，12px 内联标签

**四种来源类型的视觉编码**:

| 来源类型 | 背景色 | 文本 | 图标 |
|---------|--------|------|------|
| 资产库 (asset-library) | `#1677FF` 蓝底白字 | "资产库" | SourceAssetIcon |
| 知识库 (knowledge-base) | `#52C41A` 绿底白字 | "知识库" | SourceKnowledgeIcon |
| AI 推理 (ai-inference) | `#FAAD14` 橙底白字 | "AI 推理" | SourceAiIcon |
| 无来源 (no-source) | `#FFFBE6` 黄色高亮背景 | "无来源" | WarningIcon |
| 已编辑 (user-edited) | `#F5F5F5` 灰底灰字 | "已编辑" | EditIcon |

**交互行为**:
- 默认: 显示小型标签 (图标 + 简称)
- Hover: 标签微放大，显示完整来源类型名称
- Click: 展开 SourceDetailPopover

### 3.2 来源详情弹出框 (SourceDetailPopover)

**触发**: 点击来源标注标签
**组件**: Ant Design Popover (不打断编辑流)
**内容**:

```
┌─────────────────────────────────────┐
│ 📘 来源类型: 资产库                    │
│                                     │
│ 📂 原始出处:                          │
│   company-data/assets/产品说明.md     │
│                                     │
│ 📝 匹配片段:                          │
│ ┌─────────────────────────────────┐ │
│ │ "系统支持并发处理 1000 个请求..."   │ │
│ └─────────────────────────────────┘ │
│                                     │
│ 📊 匹配度: 87%                       │
│ ████████░░ 87%                      │
└─────────────────────────────────────┘
```

**无来源时显示**:
```
┌─────────────────────────────────────┐
│ ⚠️ 无明确来源                        │
│                                     │
│ 此段落为 AI 推理生成内容，              │
│ 未找到匹配的资产库或知识库来源。          │
│                                     │
│ 💡 建议: 请人工确认内容准确性            │
└─────────────────────────────────────┘
```

### 3.3 基线不匹配标记 (BaselineMismatchMarker)

**位置**: 与产品基线不匹配的内容段落
**视觉表现**:
- 红色下划线 (`#FF4D4F`) 标记不匹配文本
- 段落左侧显示红色警告图标
- 红色边框包裹不匹配段落

**Tooltip 内容**:
```
┌─────────────────────────────────────┐
│ ⛔ 基线不匹配                         │
│                                     │
│ 声明: "支持 5000 并发连接"             │
│ 基线: "最大支持 2000 并发连接"          │
│                                     │
│ 原因: 参数超出产品实际能力范围            │
└─────────────────────────────────────┘
```

### 3.4 章节生成后自动标注流程

```
用户点击"生成"按钮
  ↓
Story 3.4 的章节内容真正 replace 成功
  ↓
ChapterGenerationProgress 保留既有四个核心阶段
  ├── 解析需求上下文 (analyzing)
  ├── 匹配资产素材 (matching-assets，Alpha 仍为占位文案)
  ├── AI 正在撰写 (generating)
  └── 标注来源 (annotating-sources)
  ↓
若能解析到基线文件，则进度卡额外显示“基线验证”第五视觉槽位/secondary note
  ↓
编辑器段落旁逐步出现来源标签 (带淡入动画)
  ↓
若有 mismatch → 段落标红 (带闪烁提示动画，2 秒后静态)
```

### 3.5 编辑后标签降级

**触发**: 用户手动修改 AI 生成的段落内容
**行为**: 
- 对应段落的来源标签平滑过渡为灰色"已编辑"状态
- 不再关联原始来源信息，也不再展开旧来源 Popover
- 不自动重新触发来源标注 (避免频繁 AI 调用)
- 该状态由当前段落内容与持久化 digest 失配派生，而不是每次键入都回写 metadata

## 4. 页面流

### 4.1 主流程: AI 生成 → 来源标注

```
[编辑器页面]
     │
     ├── 用户点击章节"生成"按钮
     │      ↓
     ├── 生成进度条 (含"来源标注中"阶段；若存在基线则附带第 5 个视觉槽位)
     │      ↓
     ├── 内容真正写入编辑器后，来源标签出现
     │      ↓
     ├── 用户点击来源标签 → Popover 详情
     │      ↓
     └── 用户编辑内容 → 标签降级为"已编辑"
```

### 4.2 基线验证流程

```
[编辑器页面]
     │
     ├── 章节内容已写入编辑器 + 基线文件存在
     │      ↓
     ├── 自动触发基线交叉验证
     │      ↓
     ├── 不匹配项标红 + 红色图标
     │      ↓
     ├── 用户 hover 红色标记 → Tooltip 显示详情
     │      ↓
     └── 用户修改不匹配内容 → 红色标记消失
```

## 5. 视觉规格

### 5.1 来源标签

- 字号: 12px
- 行高: 18px
- 内边距: 2px 6px
- 圆角: 4px
- 图标大小: 12px
- 图标与文字间距: 4px
- 标签与段落间距: 8px (右侧边距)

### 5.2 基线不匹配标记

- 下划线: 2px 实线, `#FF4D4F`
- 警告图标: 14px, `#FF4D4F`
- 边框: 1px 实线, `#FF4D4F`, 圆角 4px
- 背景: `#FFF1F0` (极淡红)

### 5.3 Popover

- 宽度: 320px
- 内边距: 16px
- 阴影: Ant Design 默认 Popover 阴影
- 匹配片段: 引用块样式 (左侧 3px 蓝色边框)
- 匹配度进度条: 高度 6px, 圆角 3px

### 5.4 动画

- 来源标签出现: fadeIn 300ms ease-in-out
- 标签降级: 颜色过渡 200ms ease
- 基线不匹配: 边框闪烁 2 次后静态

## 6. 状态管理

### 6.1 加载状态

- 来源标注进行中: 段落旁显示骨架屏标签 (灰色闪烁占位)
- 基线验证进行中: 进度卡第 5 视觉槽位或 secondary note 显示“基线验证中”

### 6.2 错误状态

- 来源标注失败: 段落旁显示"标注失败"灰色标签 + 重试按钮
- 基线验证失败: 不显示标记，控制台日志记录

### 6.3 空状态

- 无基线文件: 不显示基线验证相关 UI
- 非 AI 生成内容: 不显示来源标签
- 章节生成进入 conflicted 且用户未确认替换: 不提前渲染来源标签或 mismatch 标记

## 7. 响应式考虑

- 编辑器宽度 < 800px: 来源标签折叠为仅图标模式
- Popover 在窗口边缘时自动调整方向 (Ant Design 内置)

## 8. 无障碍

- 来源标签含 `aria-label` 描述来源类型
- 基线不匹配标记含 `role="alert"` 
- Popover 支持 Escape 键关闭
- 颜色编码同时使用图标辅助区分 (不仅依赖颜色)
